# Магистерская диссертация — стиль и структура

Эта skill активируется при работе над текстом ВКР. Она инкапсулирует (а) семь критериев ГЭК с весами, (б) ключевые правила русского научного стиля, (в) текущее состояние артефактов проекта, чтобы текст соответствовал коду и эксперименту, а не презентации, которая местами устарела.

## 1. Что и сколько весит на защите

| Критерий | Вес | Что оценивается |
|---|---|---|
| 2. Теор. анализ + актуальность + новизна | **0.2** | ≥30 источников, лакуны, аналитический синтез |
| 3. Гипотезы + методология | **0.2** | Опровержимые H1/H2 с метриками, пошаговая методика, эксперимент воспроизводим |
| 4. Эмпирические результаты | **0.2** | Stat sig, ограничения, выводы из данных |
| 1. Проблема, цель, объект, предмет | 0.1 | Проблема ≠ область; цель ≠ тема |
| 5. Перспективы и рекомендации | 0.1 | Конкретны, опираются на полученные данные |
| 6. Структура и логика | 0.1 | Презентация поддерживает, не дублирует |
| 7. Работа с обратной связью | 0.1 | Отвечать с отсылками к данным/источникам |

**Приоритет редактуры:** критерии 2, 3, 4 = 60% оценки. Если время ограничено — править их.

## 3. Русский научный стиль — короткий чек-лист

**Глаголы и существительные:**
- Заменять глаголы в личной форме на отглагольные существительные: «мы измерили» → «измерение позволило».
- Глагольно-именные конструкции: «фильтровать» → «проводить фильтрацию».
- Глаголы — в 3 л. мн. ч. без «мы» (рассматривают, используют) или в безличной/страдательной форме (исследование проводится, было показано).
- Никогда не «я».

**Прилагательные и порядок слов:**
- Краткая форма прилагательных там, где это уместно («эффект заметен» вместо «становится заметным»).
- Прямой порядок слов (подлежащее → сказуемое → дополнение).

**Что выкинуть:**
- Эмоционально-оценочные слова: потрясающий, грандиозный, невероятный, плачевный.
- Восклицательные знаки. Вопросительные — только по делу, без вопросно-ответной формы.
- Иноязычные слова при наличии русского эквивалента: «юзер» → «пользователь», «форсировать» → «ускорить».
- Просторечие, жаргон, разговорные фразеологизмы.
- Избыточные слова: «существующая практика» → «практика»; «таким образом, исходя из всего вышесказанного» → «таким образом».

**Связочные средства (использовать активно):**
- Причинно-следственные: следовательно, в результате, вследствие, благодаря, ввиду.
- Сопоставление: с одной стороны / с другой стороны, в отличие от, наряду с.
- Связь: как было показано, согласно этому, в соответствии с.
- Вывод: таким образом, итак, в результате, обобщая сказанное.

**Доказательность:**
- Любой оценочный тезис («убедительный», «значимый») подкреплять цифрой, ссылкой, p-value или цитатой источника. Иначе — вычеркнуть.

## 4. Что должно быть в каждой секции

### Введение (критерий 1)

Обязательные блоки в этом порядке:
1. **Проблема** — конкретный разрыв в знании, обоснованный анализом теории/практики (не «область N важна»).
2. **Актуальность** — рынок/последствия/масштаб с цифрами и ссылками (см. слайд 3 презентации как рабочий шаблон).
3. **Объект** = модели (Wav2Lip, SadTalker). **Предмет** = кросс-модальная функция потерь. Не путать.
4. **Цель** = ожидаемый результат, не процесс. Глагол совершенного вида (разработать и валидировать), измеримый критерий.
5. **Гипотезы H1, H2** с количественными порогами и методом проверки (см. §5 ниже).

### Теоретический анализ (критерий 2 — 0.2)

- ≥30 источников в профильной части. Список в презентации [с.23-25] = 30 источников, можно опираться.
- НЕ пересказ, а синтез: сравнение подходов, выявление ограничений, основания для гипотез.
- Завершить таблицей сравнения (как слайд 2) с явной строкой «наш подход» и колонкой «что закрывает».
- Явно сформулировать **теоретический пробел** — одно предложение в конце раздела: «В литературе отсутствует X для Y без Z».
- **Новизна** = вклад в знание, а не «использовали SupCon». Формулировка: «Впервые показано, что [результат] достигается без [условие, требуемое предыдущими работами]».

### Методология (критерий 3 — 0.2)

Структура per-гипотеза, не per-метод:
1. **H1 → методика проверки** (4 шага со слайда 7 как костяк): калибровка энкодеров → формула лосса → файнтюнинг Wav2Lip → сравнение на test split.
2. **H2 → методика проверки** (слайд 8): использовать энкодеры из H1 → поиск веса для SadTalker → файнтюнинг → сравнение.
3. Для каждого шага: вход, операция, выход, метрика, источник данных.
4. Эксперимент должен быть **воспроизводим**: указать seed, batch size, lr, epochs, разбиение, версии моделей.
5. Артефакты (код, ноутбуки, чекпойнты) — часть методологии, со ссылками: «реализация в [04_finetune_wav2lip.ipynb], cell 7».
6. **Лосс-абляция (04b)** — это часть методологии, обосновывает выбор SupCon среди CE/cos/CE+KL. Не пропускать, иначе выбор лосса выглядит произвольным.

Гипотеза = «<условие> → <измеряемое следствие> ≥ <порог> при <ограничение> ≤ <порог>, p < 0.05 (<тест>)». Шаблон с слайда 6 — рабочий, использовать.

### Эмпирические результаты (критерий 4 — 0.2)

- Числа берутся из ноутбуков/W&B, **не из презентации** (она на старом 3-эмоция/cos прогоне).
- Per-emotion разбивка обязательна (как слайд 17). Если на 6 эмоциях — все 6, не только три «хорошо работающих».
- **Stat sig** для каждого утверждения о превосходстве: McNemar для F1 (categorical), paired t-test для LSE-C, Wilcoxon для recon. Указывать p, χ², n01/n10.
- **Внешняя валидация** (06_external_evaluation.ipynb с `dima806/...`) — отдельный подраздел: «Чтобы избежать кругового рассуждения (внутренние энкодеры использованы и в loss, и в оценке), параллельно проведена оценка независимым классификатором X». Это закрывает основное замечание рецензентов.
- **Ограничения** — отдельный подраздел, не последний абзац: 72 теста, актеры RAVDESS = одна студия, отсутствие in-the-wild видео, и т.д.
- Каждое подтверждение/опровержение гипотезы — явным боксом: «H1 подтверждена: ΔF1 = +X%, p = Y, ΔLSE-C = +Z%, p = W, оба критерия выполнены».

### Перспективы и рекомендации (критерий 5)

- Опираться на конкретные ограничения из §4: «72 теста → расширить на CREMA-D, MEAD»; «3→6→8 эмоций»; «перцепционная оценка A/B».
- **Внешняя валидация в научном поле** — упомянуть конференцию 68-я МФТИ 04.04.2026 и публикацию тезисов (https://conf.mipt.ru/conference/23659).
- Не «возможно расширить и т.д.» без связи с данными.

### Заключение

Зеркало введения: для каждой задачи — что сделано, со ссылкой на раздел. Финальный абзац — статус H1 (подтверждена) и H2 (частично, в процессе).

## 5. Типичные ошибки (по разбору ГЭК прошлых лет)

- ❌ Описание области вместо проблемы. ✅ «В литературе X отсутствует Y» с цитатой.
- ❌ Цель = тема перефразирована. ✅ Цель = измеримый результат.
- ❌ Объект и предмет путаются. ✅ Объект — что изучаем (модели), предмет — на что воздействуем (loss).
- ❌ Новизна = «применили современный метод». ✅ Новизна = «впервые показано, что …».
- ❌ Методы перечислены, методика отсутствует. ✅ Per-гипотеза пошаговый план.
- ❌ Гипотеза без измеримого порога / без теста. ✅ ΔF1 ≥ X%, p < 0.05, McNemar.
- ❌ Результаты как лог тренировки. ✅ Per-emotion + stat sig + ограничения + явный вердикт по гипотезе.
- ❌ Выводы интерпретируют, а не подтверждают/опровергают. ✅ Сначала вердикт по H, потом интерпретация.
- ❌ Перспективы декоративны. ✅ Перспектива = ответ на конкретное ограничение из §4.
- ❌ Внутренние энкодеры использованы и в loss, и в метрике (циклическая оценка). ✅ Явно описать внешнюю валидацию (06).

## 6. Что писать для H2 (SadTalker), пока он «частично подтверждён»

Не натягивать вердикт. Корректные формулировки:
- «На момент защиты H2 находится в стадии валидации; промежуточные результаты по N сэмплам показывают тренд ΔF1 = +X%, что согласуется с H1, но статистическая значимость пока не достигнута на полной выборке.»
- В перспективах: «Завершение валидации H2 с расширенной выборкой».
- В презентации/докладе озвучивать как «архитектурная универсальность подтверждена качественно, количественная валидация продолжается».

## 7. Шаблоны формулировок (готовые блоки)

**Гипотеза:**
> Включение [механизма] в [процедуру] на [данных] приведёт к статистически значимому улучшению [метрика] не менее чем на [порог]% по сравнению с [baseline], при условии, что [ограничение метрика] увеличится не более чем на [порог]%.
>
> **Критерий подтверждения:** Δ[метрика] ≥ +[X]% (p < 0.05, [тест]), Δ[ограничение] ≤ +[Y]% (p < 0.05, [тест]).

**Новизна:**
> Впервые показано, что [целевой эффект] достижим [способом] без [требование предыдущих работ — явная разметка / изменение архитектуры / etc.], что отличает предложенный подход от [группа работ A] и [группа работ B].

**Ограничение:**
> Использованы N сэмплов тестовой выборки одного датасета (RAVDESS, студийная запись 24 актёров). Это ограничивает внешнюю валидность относительно in-the-wild видео и культурно-языкового разнообразия. Для устранения требуется валидация на CREMA-D и MEAD.
